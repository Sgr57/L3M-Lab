```typescript
import { create } from 'zustand'
import { persist } from 'zustand/middleware'

interface ModelUsageState {
  lastUsed: Record<string, number>  // key: "modelId::quantization", value: timestamp ms
  setLastUsed: (modelId: string, quantization: string) => void
  getLastUsed: (modelId: string, quantization: string) => number | null
  removeUsage: (modelId: string, quantization?: string) => void
}

export const useModelUsageStore = create<ModelUsageState>()(
  persist(
    (set, get) => ({
      lastUsed: {},
      setLastUsed: (modelId: string, quantization: string): void =>
        set((state) => ({
          lastUsed: { ...state.lastUsed, [`${modelId}::${quantization}`]: Date.now() },
        })),
      getLastUsed: (modelId: string, quantization: string): number | null =>
        get().lastUsed[`${modelId}::${quantization}`] ?? null,
      removeUsage: (modelId: string, quantization?: string): void =>
        set((state) => {
          const next = { ...state.lastUsed }
          if (quantization) {
            delete next[`${modelId}::${quantization}`]
          } else {
            for (const key of Object.keys(next)) {
              if (key.startsWith(`${modelId}::`)) delete next[key]
            }
          }
          return { lastUsed: next }
        }),
    }),
    { name: 'model-usage-tracking' }
  )
)
```

Create `src/lib/cacheManager.ts` with the following exports. This is the core cache management library used by CachedModelsTable and ModelDownloader components.

**Constants:**
```typescript
const CACHE_NAME = 'transformers-cache'
const HF_URL_PATTERN = /^https:\/\/huggingface\.co\/([^/]+\/[^/]+)\/resolve\/[^/]+\/(.+)$/
```

**Export 1: `enumerateCache()`**
- Opens `transformers-cache` via `caches.open(CACHE_NAME)`
- Calls `cache.keys()` to get all Request objects
- For each request, match URL against `HF_URL_PATTERN` to extract `modelId` and `filepath`
- For each matched request, call `cache.match(req)` to get the response
- Read size from `Content-Length` header first; fall back to `(await response.clone().blob()).size` if header is missing or NaN
- Use `Promise.all()` on the array of requests for parallel processing (per RESEARCH.md Pitfall 2)
- Filter out null entries
- Return `CacheEntry[]`
- Wrap entire function in try/catch, return empty array on error

**Export 2: `quantizationFromFilepath(filepath: string): string | null`**
- Extract filename from filepath: `filepath.split('/').pop()?.toLowerCase() ?? ''`
- Match quantization using same logic as `hfSearch.ts matchQuantization`:
  - `'model.onnx'` -> `'fp32'`
  - includes `'fp16'` or `'float16'` -> `'fp16'`
  - includes `'q4'` or `'int4'` or `'bnb4'` -> `'q4'`
  - includes `'q8'` or `'int8'` or `'uint8'` or equals `'model_quantized.onnx'` -> `'q8'`
  - Otherwise return `null` (non-ONNX files like config.json, tokenizer.json)

**Export 3: `groupByModelAndQuant(entries: CacheEntry[], usageStore: { getLastUsed: (modelId: string, quantization: string) => number | null }): CachedModelInfo[]`**
- Group entries by `modelId`
- For each model, sub-group files by quantization (via `quantizationFromFilepath`)
- Files with null quantization (config.json, tokenizer.json, etc.) are counted in the model's total size but not assigned to any specific quantization group — they are "shared files"
- For each quantization group, sum the sizes and collect filepaths
- Merge `lastUsed` from the usage store using `usageStore.getLastUsed(modelId, quantization)`
- For model-level `lastUsed`, take the maximum timestamp across all quantizations (or null if none)
- For model-level `totalSize`, sum across all quantization sizes PLUS shared file sizes
- Return `CachedModelInfo[]` sorted by `modelId` ascending

**Export 4: `deleteCachedModel(modelId: string, dtype?: string): Promise<{ filesDeleted: number; filesCached: number }>`**
- Try `ModelRegistry.clear_cache()` first:
  - Import `ModelRegistry` from `'@huggingface/transformers'`
  - If `dtype` is provided, call `ModelRegistry.clear_cache(modelId, { dtype, include_tokenizer: false, include_processor: false })` (preserve shared files per RESEARCH.md Open Question 1)
  - If `dtype` is NOT provided (delete all), call `ModelRegistry.clear_cache(modelId)` (deletes everything including shared files)
  - Return `{ filesDeleted: result.filesDeleted, filesCached: result.filesCached }`
- Catch block: fall back to raw Cache API deletion:
  - Open `transformers-cache`, get all keys
  - For each key URL containing the `modelId`, optionally filter by dtype using `quantizationFromFilepath`
  - Call `cache.delete(req)` for matching entries
  - Return counts

**Export 5: `getStaleModelKeys(models: CachedModelInfo[], maxAgeMs: number): { modelId: string; quantization: string }[]`**
- Filter quantizations where `lastUsed` is non-null AND `Date.now() - lastUsed > maxAgeMs`
- Also include quantizations where `lastUsed` is null (never used = stale)
- Return array of `{ modelId, quantization }` pairs
- `maxAgeMs` will be called with `14 * 24 * 60 * 60 * 1000` (2 weeks) by the UI

Note: Use `type` import for types from `'@huggingface/transformers'` if needed, or cast the result. The `ModelRegistry` is a runtime import. If TypeScript complains about `ModelRegistry.clear_cache` return type, type the result as `any` and extract `filesDeleted`/`filesCached` properties.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser Cache API | Reading cached model data from browser-managed storage |
| HuggingFace CDN URLs | Parsing URLs from cache keys to extract model IDs |
| localStorage | Persisting usage timestamps via Zustand persist |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | cacheManager.ts URL parsing | accept | Cache API is origin-scoped (same-origin policy enforced by browser). URLs are from our own cache writes. Regex extraction is read-only. Low risk. |
| T-07-02 | Information Disclosure | useModelUsageStore localStorage | accept | Usage timestamps contain only model IDs and timestamps -- no PII, no secrets. Stored in same localStorage as existing API keys store. |
| T-07-03 | Denial of Service | enumerateCache Promise.all | mitigate | Large caches could cause memory pressure from parallel blob reads. Mitigated by preferring Content-Length header over blob reads, and wrapping in try/catch to gracefully degrade to empty array. |
| T-07-04 | Spoofing | ModelRegistry.clear_cache | accept | Deletion is user-initiated and confirmed via window.confirm(). No authentication needed for local cache operations. |
</threat_model>
