**Update `startDownload` function** to initialize the multi-model download progress in the store before posting to worker:

```typescript
export function startDownload(configs: TestConfig[]) {
  const store = useCompareStore.getState()
  const localConfigs = configs.filter((c) => c.backend !== 'api')

  // Initialize multi-model download progress
  const models: ModelDownloadStatus[] = localConfigs.map((c) => ({
    configId: c.id,
    modelName: c.displayName,
    status: 'waiting' as const,
    progress: 0,
    loaded: 0,
    total: 0,
  }))

  store.setDownloadProgress({ models, currentIndex: 0 })
  store.setExecutionStatus('downloading')

  const cmd: WorkerCommand = { type: 'download', configs: localConfigs }
  getWorker().postMessage(cmd)
}
```

Note: only send `localConfigs` to worker (filter out cloud/api configs).

**Update the `download-progress` case** in `handleWorkerEvent` to update the per-model status using `updateModelDownloadStatus`:

```typescript
case 'download-progress': {
  const { configId, progress } = event.data
  const currentStatus = progress >= 100 ? 'complete' as const : 'downloading' as const
  store.updateModelDownloadStatus(configId, {
    status: currentStatus,
    progress: event.data.progress,
    loaded: event.data.loaded,
    total: event.data.total,
  })

  // Update currentIndex to match this configId's position
  const dp = store.downloadProgress
  if (dp) {
    const idx = dp.models.findIndex((m) => m.configId === configId)
    if (idx >= 0 && idx !== dp.currentIndex) {
      store.setDownloadProgress({ ...dp, currentIndex: idx })
    }
  }
  break
}
```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | ParamInput number inputs | mitigate | HTML min/max attributes constrain range (temp 0-2, maxTokens 1-4096, topP 0-1, repeatPenalty 1-2). React controlled inputs prevent non-numeric values. Worker and cloud APIs impose their own limits. |
| T-03-02 | Tampering | Store persist hydration | accept | Parameters loaded from localStorage could be tampered with via devtools. Risk is low -- this is a local-only POC with no server-side trust. Worst case: invalid param values cause a model error that is already handled. |
| T-03-03 | Denial of Service | Download cancel race | accept | Cancelling mid-download leaves the current model finishing. The pipeline() call is not abortable. Acceptable for POC -- user just waits for current file to finish. |
</threat_model>
