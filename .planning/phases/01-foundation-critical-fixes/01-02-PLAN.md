**Change 1: Replace the TextStreamer construction (lines 173-189)**

Replace the current TextStreamer constructor call with:

```typescript
const streamer = new TextStreamer(generator.tokenizer, {
  skip_prompt: true,
  token_callback_function: (_tokens: bigint[]) => {
    if (tokenCount === 0) {
      firstTokenTime = performance.now() - genStart
    }
    tokenCount++
    const elapsed = performance.now() - genStart
    const tokPerSec = tokenCount > 0 ? (tokenCount / (elapsed / 1000)) : 0

    postProgress(
      config, currentIndex, totalModels, 'generating',
      tokenCount, tokPerSec, loadTime + initTime + elapsed, streamedText
    )
  },
  callback_function: (text: string) => {
    streamedText += text
  },
})
```

Key differences from current code:
- `token_callback_function` replaces the token counting logic that was inside `callback_function`
- Parameter is `_tokens: bigint[]` (prefixed with underscore since we only need the count, not the token IDs)
- TTFT measurement moves to `token_callback_function` (fires on actual first token, not first text flush -- this is MORE accurate per Research Pitfall 3)
- `callback_function` now ONLY accumulates `streamedText` (no counting, no TTFT, no progress posting)
- Progress posting moves to `token_callback_function` (so it fires per actual token)
