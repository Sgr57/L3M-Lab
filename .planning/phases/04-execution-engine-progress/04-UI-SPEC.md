## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline padding (`py-0.5`, `px-1.5`) |
| sm | 8px | Compact element spacing (`gap-2`), inner card padding offsets |
| md | 16px | Default element spacing (`gap-4`), standard card padding (`p-4`) |
| lg | 20px | Card content padding (`p-5` = 20px, established pattern) |
| xl | 24px | Section gaps (`gap-6`), section header margins |
| 2xl | 32px | Nav horizontal padding (`px-8` = 32px) |

Exceptions:
- 12px (`mb-3`, `gap-3`) is used extensively between section header and content, between stats rows, and between card sub-sections. This is an established project convention.
- 14px (`py-3.5`, `p-3.5`) is used for output card inner padding. Inherited from Phase 2 OutputComparison.

## Component Inventory

### Modified Components

#### TestProgress (`src/components/TestProgress/index.tsx`)

**Current state:** Shows model name, phase label, token count, tok/s, elapsed time, simple progress bar, streaming text preview.

**Changes for Phase 4:**

1. **Weighted progress bar** — Replace `(currentIndex + 1) / totalModels` with weighted formula per D-01. Loading=10%, initializing=10%, generating=80% of each model's slice. Generating portion tracks `tokensGenerated / maxTokens`.

2. **Cloud model timer** — When `phase` is `'cloud-pending'`, show "Waiting for response..." text with a live elapsed timer updating every 100ms. Timer is component-local state via `setInterval` (not store-driven, per RESEARCH.md anti-pattern guidance). Display format: `0.0s`, `1.2s`, `12.3s`.

3. **Backend type badge** — Add a small colored badge next to model name showing backend type:
   - Cloud: `bg-cloud-bg text-cloud` with label "cloud"
   - WebGPU: `bg-webgpu-bg text-primary` with label "webgpu"
   - WASM: `bg-wasm-bg text-wasm` with label "wasm"
   - Pattern: Reuse `TypeBadge` from OutputComparison (10px semibold, rounded-md, px-1.5 py-0.5)

4. **Phase labels extended:**
   | Phase | Label |
   |-------|-------|
   | `loading` | "Loading model..." |
   | `initializing` | "Initializing..." |
   | `generating` | "Generating..." |
   | `disposing` | "Cleaning up..." |
   | `cloud-pending` | "Waiting for response..." |
   | `cloud-complete` | "Complete" |

5. **Progress bar visual continuity** — Keep existing gradient (`bg-gradient-to-r from-primary to-[#218bff]`) and transition (`transition-all duration-300`). The weighted formula provides smooth movement; no additional animation needed.

#### ComparePage (`src/pages/ComparePage.tsx`)

**Changes for Phase 4:**

1. **FallbackBanner placement** — Render `<FallbackBanner />` between `<TestProgress />` and the results section. Visible only when `fallbackWarning` state is non-null. Persists until run completes (D-05).

2. **Error results inline** — No layout change needed. Error results with `errorCategory` are added to `results[]` via `addResult()` in execution order. OutputComparison already renders error cards with red tint. Phase 4 enhances the error display within the existing card pattern.

### New Components

#### FallbackBanner (`src/components/FallbackBanner/index.tsx`)

**Purpose:** Persistent warning banner shown when WebGPU device loss triggers WASM fallback (D-04, D-05).

**Visual spec:**
- Container: `rounded-xl border border-warning/30 bg-warning/5 p-4`
- Layout: Single row, icon + text
- Icon: Inline SVG warning triangle, 16x16, `text-warning`
- Text: 13px regular, `text-text-primary`
- Copy: "WebGPU device lost -- remaining models switched to WASM"
- No dismiss button (persists until run completes per D-05)
- Disappears when `executionStatus` returns to `'idle'` or `'complete'`

**State source:** `useCompareStore((s) => s.fallbackWarning)` -- renders when non-null.

#### Enhanced Error Card (within OutputComparison)

**Purpose:** Show classified cloud API errors inline with results (D-10, D-11).

**Visual spec:**
- Container: Same output card pattern but with `border-l-error` (3px left border in `#cf222e`)
- Background tint: `bg-error/5` on the card body
- Error category badge: `rounded-md px-1.5 py-0.5 text-[10px] font-semibold bg-error/10 text-error` with category label (e.g., "CORS Blocked", "Auth Failed", "Rate Limited", "Timeout", "Server Error", "Unknown Error")
- Hint text: 13px regular, `text-text-secondary`, below the category badge
- Collapsible raw error: Button "Show raw error" / "Hide raw error" in 11px `text-primary` font-medium. Expanded content in `rounded-lg bg-bg border border-border p-3 text-[13px] font-mono text-text-secondary` with word-wrap.

**Error category display labels:**

| Category | Badge Label | Hint Template |
|----------|------------|---------------|
| `cors` | CORS Blocked | "Anthropic API has CORS restrictions. A browser proxy may be needed." / "Network error reaching {provider} API. Check your connection or CORS settings." |
| `auth` | Auth Failed | "Check your {provider} API key in Settings." |
| `rate-limit` | Rate Limited | "{provider} rate limit exceeded. Wait a moment and try again." |
| `timeout` | Timeout | "{provider} request timed out. Try a shorter prompt or try again." |
| `server` | Server Error | "{provider} server error. Try again later." |
| `unknown` | Unknown Error | "An unexpected error occurred." |

#### Fallback Indicator (within OutputComparison result cards)

**Purpose:** Show "WebGPU -> WASM" when a model fell back from WebGPU to WASM (D-06).

**Visual spec:**
- Position: Next to the backend/quant metadata line in result cards
- Format: `text-[11px] text-warning font-medium` with text "WebGPU -> WASM"
- Appears only when `result.fallbackBackend` is set

## Interaction Contracts

### Progress Bar Behavior

| Scenario | Behavior |
|----------|----------|
| Local model: loading phase | Bar advances to `completedModels/total + (1/total * 0.10)` |
| Local model: initializing phase | Bar at `completedModels/total + (1/total * 0.20)` |
| Local model: generating phase | Bar smoothly fills from 20% to 100% of model slice, tracking `tokensGenerated / maxTokens` |
| Local model: disposing phase | Bar at 100% of model slice (same as next model's 0%) |
| Cloud model: pending | Bar stays at `completedModels/total` (no intermediate progress) |
| Cloud model: response received | Bar jumps to `(completedModels + 1) / total` instantly |
| All models complete | Bar at 100% |

### Cloud Timer Behavior

| Scenario | Behavior |
|----------|----------|
| Cloud call starts | Timer starts at "0.0s", updates every 100ms |
| Cloud call in-flight | Timer counts up: "0.1s", "0.2s", ..., "3.4s" |
| Cloud response arrives | Timer stops, phase changes to "Complete", metrics display |
| Cloud call errors | Timer stops, error card appears in results |
| Run cancelled during cloud call | Timer stops, execution halts |

### GPU Fallback Behavior

| Scenario | Behavior |
|----------|----------|
| Device loss during model load | FallbackBanner appears. Failed model retries on WASM. Remaining WebGPU models switch to WASM. |
| Device loss during generation | FallbackBanner appears. Failed model retries on WASM. Remaining WebGPU models switch to WASM. |
| Fallback retry succeeds | Result card shows "WebGPU -> WASM" indicator. Metrics reflect WASM execution. |
| Fallback retry also fails | Error result card appears inline. Execution continues to next model. |
| Run completes after fallback | FallbackBanner remains visible until next run or reset. |

### Error Display Behavior

| Scenario | Behavior |
|----------|----------|
| Cloud API returns 401/403 | Error card inline with "Auth Failed" badge + hint. Raw error collapsible. |
| Cloud API returns 429 | Error card inline with "Rate Limited" badge + hint. |
| Cloud API fetch throws TypeError | Error card inline with "CORS Blocked" badge + provider-specific hint. |
| Cloud API returns 5xx | Error card inline with "Server Error" badge + hint. |
| Unknown error | Error card inline with "Unknown Error" badge + generic hint. |
| Multiple errors in same run | Each error appears as its own card in execution order among successful results. |
